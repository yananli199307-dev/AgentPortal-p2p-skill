## 二、聊天功能参考（TIMUIKitChat）

### 2.1 消息发送类型

腾讯 IM 支持的消息类型及我们可以参考的：

| 消息类型 | 描述 | 参考价值 | P2P 适配方案 |
|---------|------|---------|------------|
| 文本 | 纯文字 | ✅ 已有 | — |
| 图片 | 缩略图+原图+大图三档 | 🟡 参考 | 上传文件后发 file_url |
| 视频 | 封面+视频文件 | 🟢 远期 | 同上 |
| 语音 | AMR 格式+时长 | 🟢 远期 | 录制→上传→发 URL |
| 文件 | 文件名+大小+图标 | ⚠️ 有基础 | 缺文件图标匹配 |
| 位置 | 经纬度+地图截图 | 🟢 远期 | 高德地图 SDK |
| 自定义 | JSON 自由格式 | ✅ 可扩展 | message_type=自定义 |

### 2.2 消息互动功能

腾讯 IM 的聊天交互功能及我们的改进建议：

| 功能 | 描述 | 优先级 | 实现要点 |
|------|------|--------|---------|
| **消息引用/回复** | 长按消息→引用→回复时带原文摘要 | 🔴 高 | Message 加 `reply_to` 字段 |
| **消息表情回应** | 对消息点 👍😂❤️😮😢😡 | 🔴 高 | Message 加 `reactions` JSON 数组 |
| **已读回执** | 群聊看谁读了消息 | 🟡 中 | 群成员 `read_by` 列表 |
| **消息转发** | 长按→转发到其他聊天 | 🟡 中 | 新建转发消息 |
| **消息撤回** | 发送后 n 分钟内可撤回 | 🟡 中 | Message 加 `is_revoked` + 通知 |
| **多选/删除** | 批量选择→删除 | 🟢 低 | 前端长按多选模式 |
| **翻译** | 长按→翻译 | 🟢 低 | 接入翻译 API |

| 交互 | 描述 | 建议 |
|------|------|------|

| 发送按钮 | 输入框右侧发送按钮 | ✅ 已有 |

## 七、接口设计参考

### 7.1 消息格式扩展

当前 Message 结构：
```json
{
  "id": 1,
  "content": "消息内容",
  "message_type": "text",
  "file_url": null,
  "sender_portal": "https://agentp2p.cn",
  "created_at": "2026-04-27T12:00:00"
}
```

建议扩展字段（参照腾讯 IM）：
```json
{
  "reply_to": 12,              // 引用消息 ID
  "reactions": [               // 表情回应
    {"emoji": "👍", "count": 3, "reactors": ["portalA", "portalB"]}
  ],
  "is_revoked": false,         // 是否已撤回
  "is_read": true,             // 单聊已读
  "read_by": ["portalA"],      // 群聊已读列表
  "link_preview": {            // 链接预览
    "title": "页面标题",
    "description": "摘要",
    "image": "https://..."
  },
  "extra": {}                   // 自定义扩展
}
```

### 7.2 群组类型扩展

```diff
# Group 模型
+ group_type: "work_group" | "public_group" | "chatroom"
+ announcement: str            # 群公告
+ avatar_url: str              # 群头像
```

### 7.3 回调/Webhook 参考

腾讯 IM 的回调体系可参考用于 P2P 通知：

| 事件 | 回调时机 | 我们对应 |
|------|---------|---------|
| 发消息前回调 | 消息入库前 | 可加内容审核 |
| 发消息后回调 | 消息入库后 | 我们的 P2P 推送 |
| 创建群前回调 | 群入库前 | 可限制数量 |
| 成员入群后回调 | 成员加入后 | 我们的群更新通知 |
| 好友申请回调 | 申请到达时 | 我们的 /contact-requests |
| 在线状态变更 | 上线/下线 | WebSocket 心跳 |

## 九、关键设计原则

> 以下是我们从腾讯 IM 借鉴的**设计原则**，不是代码：

1. **组件化拆分** — 每个功能独立组件，可组合可替换
2. **消息扩展字段** — 用 JSON extra 而非加列，灵活扩展
3. **长按菜单模式** — 消息/联系人/群组都支持长按操作
4. **状态同步** — 撤回/已读/反应 通过 P2P 消息实时同步
5. **离线优先** — 消息先入库再推送，保证不丢
6. **渐进增强** — 先做核心（文字+文件），再加高级功能（引用+表情+已读）
